# Weekly Review Checklist (Collector App)

## 1) เป้าหมายประจำสัปดาห์
- [ ] กำหนดเป้าหมายจำนวนเนื้อหาประจำสัปดาห์
- [ ] กำหนดหมวดหลักที่ต้องโฟกัสในสัปดาห์นี้
- [ ] ยืนยันเกณฑ์ขั้นต่ำก่อนอัปเว็บจริง (ต้องผ่าน human review)

## 2) ภาพรวมผลงานสัปดาห์นี้
- [ ] จำนวนที่นำเข้า (import/collect): ______
- [ ] จำนวนที่ผ่าน quality: ______
- [ ] จำนวนที่ approved โดยทีม: ______
- [ ] จำนวนที่ publish/stage/export สำเร็จ: ______
- [ ] จำนวนที่อัปขึ้น public website แบบ manual: ______
- [ ] เทียบกับเป้าหมาย: [ ] ถึงเป้า [ ] ต่ำกว่าเป้า [ ] เกินเป้า

## 3) ตรวจคุณภาพ input
- [ ] แหล่งข้อมูลมีความน่าเชื่อถือพอหรือไม่
- [ ] ข้อมูลซ้ำเกิดบ่อยหรือไม่
- [ ] มีรายการข้อมูลไม่ครบจนใช้งานไม่ได้กี่รายการ
- [ ] คุณภาพรูปจากแหล่งข้อมูลพอใช้งานหรือไม่
- [ ] มีหมวดหมู่/ภาษา source ผิดบ่อยแค่ไหน

## 4) ตรวจคุณภาพ draft/content
- [ ] draft title อ่านแล้วตรงประเด็น
- [ ] เนื้อหา body มีโครงสร้างอ่านง่าย
- [ ] meta title / meta description ใช้งานได้จริง
- [ ] มีรายการที่ต้องแก้มือจำนวนมากหรือไม่
- [ ] สัดส่วนงานที่ผ่านรอบแรกอยู่ในระดับที่ยอมรับได้

## 5) ตรวจ human workflow
- [ ] review queue ใช้งานคล่องหรือไม่
- [ ] ทีมเข้าใจสถานะงานแต่ละขั้นชัดเจน
- [ ] จุดไหนที่ใช้เวลาตรวจมากที่สุด
- [ ] เกณฑ์ approve/reject สอดคล้องกันในทีม
- [ ] มีขั้นตอนไหนที่ควรลดงานซ้ำ

## 6) ตรวจ translation
- [ ] translation ทำงานเฉพาะ final stage ตามที่กำหนด
- [ ] ภาษาที่ผ่าน automatic checks มีประโยชน์ต่อการใช้งานจริง
- [ ] มีภาษาไหน fail ซ้ำบ่อย
- [ ] translation ที่ไม่ผ่านถูกกันออกจาก export ถูกต้อง
- [ ] source content ยังส่งออกได้แม้บางภาษาไม่ผ่าน

## 7) ตรวจ publish / stage / export
- [ ] publish สำเร็จตามจำนวนที่ approved
- [ ] stage สะท้อนข้อมูลล่าสุดถูกต้อง
- [ ] export ไฟล์ครบและเปิดได้จริง
- [ ] ไม่มีความสับสนระหว่างไฟล์ที่จะใช้งานจริงกับไฟล์ที่ยังไม่พร้อม
- [ ] ขั้นตอน manual upload ไป public website ชัดเจนและตรวจสอบย้อนหลังได้

## 8) ตรวจปัญหาที่เจอบ่อย
- [ ] ปัญหาข้อมูลซ้ำ
- [ ] ปัญหาเนื้อหาไม่ครบ/ไม่ชัด
- [ ] ปัญหา draft คุณภาพต่ำ
- [ ] ปัญหา translation fail
- [ ] ปัญหา export หรือไฟล์ไม่พร้อมใช้งาน
- [ ] ปัญหาอื่น ๆ: __________________

## 9) สรุป pain points สัปดาห์นี้
- [ ] Pain point 1: __________________
- [ ] Pain point 2: __________________
- [ ] Pain point 3: __________________
- [ ] ผลกระทบหลักต่อทีม: __________________

## 10) สิ่งที่ควรปรับสัปดาห์หน้า
- [ ] ปรับเรื่อง input/source: __________________
- [ ] ปรับเกณฑ์หรือวิธี review: __________________
- [ ] ปรับการคัดเลือกรูปจาก media library: __________________
- [ ] ปรับการจัดลำดับงานใน pipeline: __________________
- [ ] ผู้รับผิดชอบแต่ละข้อชัดเจนแล้ว
