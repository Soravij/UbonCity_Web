# Daily Operating Checklist (Collector App)

## 1) เป้าหมายประจำวัน
- [ ] กำหนดเป้าหมายวันนี้: `2–5` เนื้อหา
- [ ] ระบุหมวดที่จะทำวันนี้ (เช่น ที่เที่ยว / คาเฟ่ / ร้านอาหาร / อีเวนต์)
- [ ] ยืนยันว่าทุกเนื้อหาต้องมีคนตรวจทานก่อนอัปขึ้นเว็บจริง

## 2) ก่อนเริ่มงาน
- [ ] เปิดระบบ `collector-app` และล็อกอินด้วยบัญชีที่ถูกต้อง
- [ ] ตรวจว่าโฟลเดอร์ `staging` และ `export` พร้อมใช้งาน
- [ ] ตรวจว่าระบบไม่ค้างจากงานเมื่อวาน (ไม่มี error ค้าง)
- [ ] ยืนยันกติกาวันนี้: AI ช่วยร่าง แต่ไม่เผยแพร่ขึ้นเว็บจริงอัตโนมัติ

## 3) นำเข้าข้อมูล
- [ ] นำเข้าข้อมูลผ่าน Import (JSON/CSV) หรือกรอกมือ
- [ ] ตรวจชื่อเรื่อง/คำอธิบายเบื้องต้นให้พออ่านรู้เรื่อง
- [ ] ลบรายการซ้ำหรือรายการที่ข้อมูลไม่พอใช้
- [ ] ตั้งหมวดหมู่และภาษา source ให้ถูกต้อง
- [ ] เลือกรูปภาพจาก Media Library แบบ manual (ยังไม่ยึดรูปอัตโนมัติ)

## 4) Run pipeline
- [ ] รัน `clean`
- [ ] รัน `ai-draft`
- [ ] รัน `quality`
- [ ] ตรวจผลลัพธ์แต่ละขั้นว่าไม่มีรายการ error ผิดปกติ
- [ ] ถ้ามีรายการไม่ผ่าน quality ให้แก้ไขและวนรอบใหม่เฉพาะรายการนั้น

## 5) Human review
- [ ] เปิด review queue และตรวจทีละรายการ
- [ ] ตรวจความถูกต้องของชื่อเรื่อง, เนื้อหา, meta title, meta description
- [ ] ตรวจความสมเหตุสมผลของข้อมูลสถานที่/อีเวนต์
- [ ] ตรวจความซ้ำกับเนื้อหาเดิม
- [ ] อนุมัติเฉพาะรายการที่พร้อมใช้งานจริง
- [ ] รายการที่ยังไม่ดีให้ `request changes` หรือ `reject`

## 6) Publish / stage / export
- [ ] รัน `publish` (เฉพาะรายการที่ approved แล้ว)
- [ ] รัน `stage`
- [ ] รัน `export`
- [ ] ตรวจว่า translation ทำงานเฉพาะ final stage เท่านั้น
- [ ] ใช้เฉพาะ translation ที่ผ่าน automatic checks (passed + non-stale)
- [ ] ยืนยันว่า source content ยัง export ได้ แม้บางภาษาแปลไม่ผ่าน

## 7) ก่อนอัปขึ้นเว็บจริง
- [ ] เปิดไฟล์ export ที่จะใช้จริง และคัดเฉพาะรายการที่พร้อม
- [ ] ตรวจอีกครั้งว่าทุกรายการมีคนตรวจแล้ว
- [ ] ตรวจ slug / meta / หมวดหมู่ / ภาพปก ก่อนนำเข้าเว็บหลัก
- [ ] ยืนยันว่า “ไม่อัปทั้งหมดทั้งก้อน” ถ้ายังมีรายการเสี่ยง
- [ ] อัปโหลดขึ้น public website แบบ manual ตามรายการที่คัดแล้ว

## 8) หลังจบงาน
- [ ] บันทึกจำนวนที่ import / approved / exported / uploaded
- [ ] สรุปรายการที่ต้องแก้ต่อพรุ่งนี้
- [ ] เก็บ log ปัญหา (เช่น แปลไม่ผ่าน, quality fail, ข้อมูลซ้ำ)
- [ ] ตรวจความเรียบร้อยของไฟล์ export ประจำวันที่ทำงาน
