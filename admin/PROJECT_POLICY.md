# Admin Project Policy

See [../PROJECT_POLICY.md](../PROJECT_POLICY.md) for the canonical project-wide policy.

## Admin-specific rules

- Admin may review resolved taxonomy data and returned values.
- Admin must not generate a second conflicting live checklist for an issued assignment.
- Category remains owned by the upstream Clean workflow unless an explicit correction workflow exists.
- No admin workflow should reinterpret historical assignment questions retroactively.
